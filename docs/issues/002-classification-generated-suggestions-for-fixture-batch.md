# Title

Classification-generated suggestions for one fixture batch

## Type

AFK

## User-visible goal

Let the user review one fixture-backed batch whose suggested labels, near-misses, confidence band, and interpretation are generated by the classification flow rather than hand-authored in the fixtures.

## Scope

- Reuse the fixture-backed review loop from issue `001`
- Start from sample message fixtures, not prebuilt review items
- Generate review-ready suggestions for each sample message: interpretation, applied labels, near-misses, confidence band, and `unlabeled` when appropriate
- Apply the preflight compatibility rules and 3-label cap before suggestions reach review
- Preserve the review-loop behavior, frozen reviewed state, and minimal batch summary already proven in issue `001`
- Define expected behavior and tests before implementation begins

## Non-goals

- real inbox access
- Gmail label creation or write-back
- training or tuning infrastructure
- broad classifier framework work
- sender-level policy

## Acceptance criteria

- A fixture batch of sample messages can be transformed into review-ready suggestions without any external inbox dependency
- The generated output exposes interpretation, applied labels, near-misses, confidence band, and `unlabeled` outcomes in the shape expected by the review loop
- Compatibility rules and the 3-label cap are enforced before the reviewer sees applied labels
- The review loop from issue `001` can consume generated suggestions without changing its user-visible semantics
- The slice includes a written expected-behavior and test list before code implementation starts

## Expected tests or verification

- Fixture-driven test for suggestion shaping from sample messages into review-ready items
- Test that `unlabeled` is produced when no label should be applied
- Test that over-cap or incompatible label combinations are reshaped into applied labels plus near-misses
- Test that generated confidence bands and interpretation are exposed through the public review data contract
- Manual verification on one representative fixture batch with generated suggestions

## Dependencies/order

- Depends on issue `001`
- Should start only after `docs/decisions/review-semantics.md` is approved

## Stop conditions requiring Founder review

- The slice appears to require taxonomy expansion or changed review semantics
- Useful generated suggestions seem to require real inbox context rather than fixture/sample messages
- The classification flow pressures the project toward broad model infrastructure instead of a narrow slice
