# Friendly Companion Error State

GitHub issue: `#40`

## What to build

Improve disconnected/error presentation in the Gmail companion sidebar so it gives a clear product status, useful recovery steps, technical detail behind an expander, and a retry action.

## Acceptance criteria

- [x] Error state shows friendly failure copy before technical detail.
- [x] Technical detail remains available behind an expandable section.
- [x] `Check again` forces a fresh state request.
- [x] Tests protect the recovery action markers.
