The compressed mental model: a test is evidence only for the path it actually exercised.

Four questions

1. Environment: where did the test run?
2. Entry point: where did it begin?
3. Actions: what was actually clicked or typed?
4. Evidence: what proves each step passed?
Resume-safe explanation: “I distinguish logic tests, controlled browser acceptance, live-service harnesses, and normal user click-through. I match each product claim to evidence from the path that actually exercises it.”
Threadwise safety: live Gmail testing touches a sensitive surface and requires explicit founder approval. A blocked live test must be reported as BLOCKED, not replaced silently with simulator evidence.
